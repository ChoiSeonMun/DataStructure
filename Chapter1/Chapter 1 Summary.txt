1. 자료구조와 알고리즘이란?
>> 자료구조는 자료를 정리하고 조직화하는 구조를 일컬으며, 알고리즘은 문제 해결 방법을 말한다.

2. 알고리즘의 표기 방법엔 어떤 것들이 있는가?
>> 자연어로 기술, 특정 프로그래밍 언어로 기술, 의사코드, 흐름도가 있다

3. 추상화는 왜 필요한가?
>> 사용자에게 소프트웨어의 복잡성을 숨기고, 핵심적인 기능만 간추려 편리함을 제공한다.

4. 추상 자료형이란?
>> 해당 자료형의 자료와 연산의 집합을 말한다.

5. 알고리즘 실행 시간은 어떻게 측정하는가?
>> 알고리즘을 실제 구현하여 실행 시간을 측정하거나, 복잡도 분석을 이용해 측정한다.

6. 알고리즘의 시간 복잡도는 무엇인가?
>> 알고리즘의 실행 시간이 얼마나 걸리는 지를 의미한다. 알고리즘을 이루고 있는 연산들이 몇 번이나 실행되는지를 숫자로 표시한다.

7. 빅오 표기법은 어떻게 표기하는가?
>> 실행 시간에 가장 큰 영향을 미치는 연산 갯수의 최고차항을 표기한다.

8. 자료구조는 어떻게 표기하는가?
>> UML을 이용한다. 자료구조의 이름, 자료명, 연산 순으로 쓰며, 콜론(:)을 이용해 연산의 반환형이나 자료의 타입을 쓰고, +, -, #을 이용해 접근 수준을 나타낸다.
